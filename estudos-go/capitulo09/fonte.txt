Capítulo 9: Ponteiros - O Endereço da Casa


Introdução: A Foto vs. A Casa

Bem-vindo à Parte 2. Aqui separamos os amadores dos profissionais. Muitos estudantes tremem ao ouvir a palavra "Ponteiro". Em linguagens como C, eles são perigosos. Em Go, eles são amigáveis, mas você precisa entender o conceito fundamental de Valor vs Referência.

Imagine que você tem uma casa linda (sua variável) e quer mostrá-la para um amigo (uma função). Você tem duas opções:

1. Por Valor (Cópia): Você tira uma foto da casa e entrega a foto para o amigo.
   - Se o amigo desenhar um bigode na foto, sua casa real muda? Não.
   - Isso é seguro, mas se a casa for gigantesca (cheia de móveis), "tirar a foto" (copiar todos os dados) demora e gasta filme (memória).

2. Por Referência (Ponteiro): Você anota o endereço da casa num papel (Rua dos Bobos, 0) e entrega o papel.
   - O amigo pega o carro, vai até o endereço, entra na casa e pinta a parede de roxo.
   - Sua casa real mudou? Sim!
   - Isso é rápido (passar um papelzinho é leve), mas perigoso (o amigo pode bagunçar sua casa de verdade).

Um Ponteiro nada mais é do que esse papel com o endereço. Ele não guarda o dado. Ele guarda onde o dado está.


---


1. Os Operadores Mágicos: & e *

Em Go, usamos dois símbolos para brincar com endereços.

O "E" Comercial (&): "Onde você mora?"

Se você tem uma variável x, ao colocar &x, você está perguntando ao Go: "Qual é o endereço de memória da variável x?". Isso gera um ponteiro.

	idade := 30       // Variável normal (A casa)
	endereco := &idade  // O ponteiro (O papel com o endereço)

O tipo de endereco não é int. É *int (lê-se: "Ponteiro para inteiro").

O Asterisco (*): "O teletransporte"

Se você tem um endereço na mão, você usa o * para "viajar" até lá e ver o que tem dentro. Chamamos isso de Desreferenciar (Dereferencing).

	fmt.Println(*endereco)  // Viaja até o endereço e imprime: 30
	*endereco = 31          // Viaja até o endereço e muda o valor original!


---


2. Por que usar Ponteiros?

Existem dois motivos principais:

1. Mutabilidade (Efeito Colateral): Quando você passa uma variável para uma função em Go, ela recebe uma Cópia. Se você quer que a função altere a variável original, você precisa passar um ponteiro.

	// Recebe por CÓPIA (Foto)
	func tentarMudar(x int) {
		x = 99  // Só riscou a foto. O x original continua intacto.
	}

	// Recebe por PONTEIRO (Endereço)
	func mudarDeVerdade(x *int) {
		*x = 99  // Viajou até a memória e alterou o valor real.
	}

2. Performance (Eficiência): Imagine uma Struct gigante (veremos no próximo capítulo) que representa um arquivo de vídeo de 1GB. Cada vez que você passar esse vídeo para uma função, o Go vai tentar copiar 1GB de memória. Isso trava o computador. Se você passar um Ponteiro (o endereço), você só copia 8 bytes (o tamanho de um endereço de memória, que é minúsculo). É instantâneo.


---


3. O Perigo do nil (Nulo)

Endereços podem apontar para lugar nenhum. O Zero Value de um ponteiro é nil.

Se você declarar um ponteiro mas não colocar nenhum endereço nele:

	var p *int  // p é nil

Se você tentar viajar para um endereço nulo (*p = 10), o programa entra em Panic (Erro fatal: "invalid memory address or nil pointer dereference"). É como tentar entrar numa casa num endereço que não existe no mapa. Você cai no abismo.

Sempre verifique se o ponteiro não é nulo antes de usar, se houver dúvida.


---


4. new vs make

Você vai ver uma função chamada new(Tipo). p := new(int) Ela faz três coisas:

1. Aloca memória para um int (valor 0).
2. Gera o endereço dessa memória.
3. Retorna o ponteiro.

É equivalente a:

	var i int  // Cria variável zero
	p := &i    // Pega endereço

Diferença Ninja:
- make: Só para Slices, Maps e Channels (coisas complexas que precisam de inicialização interna). Retorna o objeto em si (não ponteiro).
- new: Para tipos básicos e Structs. Retorna Ponteiro.


---


Dica do Gopher

[TIP] Não seja o "Maníaco do Ponteiro"

Vindo de C++, as pessoas tendem a usar ponteiros para TUDO em Go para "economizar memória". Não faça isso. O compilador Go é inteligente. Copiar pequenas structs ou variáveis básicas (int, bool) é extremamente rápido, muitas vezes mais rápido que ponteiros (devido a como a CPU funciona com cache).

Regra de Ouro:
- Use Ponteiros se você precisa compartilhar ou modificar o estado.
- Use Ponteiros se a estrutura for grande.
- Para todo o resto, use Valores. É mais seguro e fácil de ler.

Exemplos Práticos

Pasta capitulo09, arquivo main.go.

package main

import "fmt"

// Função que recebe uma CÓPIA (Valor)
func zerarValor(x int) {
	x = 0  // Altera apenas a cópia local
}

// Função que recebe um ENDEREÇO (Ponteiro)
func zerarPonteiro(x *int) {
	*x = 0  // Vai até a memória original e zera
}

func main() {
	// 1. Entendendo Endereços
	numero := 42
	var ponteiro *int = &numero

	fmt.Println("Valor de numero:", numero)
	fmt.Println("Endereço de numero:", ponteiro)      // Algo como 0xc0000...
	fmt.Println("O que tem no endereço:", *ponteiro)

	// 2. Mudando valor via ponteiro
	*ponteiro = 100  // Alterando indiretamente
	fmt.Println("Novo valor de numero:", numero)  // Mágica! Mudou para 100.

	// 3. Teste Valor vs Referência
	a := 10
	b := 10

	zerarValor(a)
	zerarPonteiro(&b)  // Passamos o endereço de b (&b)

	fmt.Println("A (Valor):", a)      // Continua 10
	fmt.Println("B (Ponteiro):", b)   // Virou 0

	// 4. Ponteiro de Ponteiro (Inception) - Só por curiosidade
	// Ponteiro 'p' aponta para 'numero'
	// Ponteiro 'pp' aponta para o ponteiro 'p'
	var pp **int = &ponteiro
	fmt.Println("Endereço do ponteiro:", pp)
	fmt.Println("Valor lá no fundo:", **pp)  // 100
}

Exercícios

1. O Trocador (Fácil)

Escreva uma função trocar(a, b *int) que recebe dois ponteiros para inteiros e troca os valores deles. Na main: x := 5, y := 10. Chame trocar(&x, &y). fmt.Println(x, y) deve mostrar 10 5. Dica: Você vai precisar de uma variável temporária ou usar o truque de múltipla atribuição do Go: *a, *b = *b, *a.


2. O Incrementador (Médio)

Crie uma variável contador := 0. Crie uma função incrementar que aceite um ponteiro e adicione +1 ao valor. Faça um loop que chame essa função 10 vezes passando o endereço do contador. Imprima o resultado final (deve ser 10).


3. O Detetive de Endereços (Conceitual)

Crie uma variável texto := "Go". Imprima o endereço dela (&texto). Chame uma função passando texto (por valor). Dentro da função, imprima o endereço do parâmetro recebido. Eles são iguais ou diferentes? Por quê? (Lembre da analogia da Foto).

func verEndereco(t string) {
	fmt.Println("Endereço dentro da função:", &t)
}

Isso provará para você que uma cópia foi criada em outro lugar da memória.
